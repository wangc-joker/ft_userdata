# NostalgiaForInfinityX7 策略详解 (初学者教程)

## 一、这是什么？

`NostalgiaForInfinityX7` 是一个基于 **Freqtrade** 框架编写的加密货币自动交易策略 (版本 v17.4.7)。Freqtrade 是一个开源的加密货币交易机器人框架，而这个策略是社区里最知名、最复杂的策略之一，由开发者 `iterativ` 编写。

**打个比喻**：如果 Freqtrade 是一辆汽车的"底盘和引擎"，那 NostalgiaForInfinityX7 就是"驾驶员的大脑"，它告诉机器人**什么时候买、什么时候卖、买多少**。

---

## 二、基本运行参数 (第 68~200 行)

```
类: NostalgiaForInfinityX7(IStrategy)   # 继承 Freqtrade 的策略接口
版本: INTERFACE_VERSION = 3
时间周期: timeframe = "5m"               # 主时间周期是 5 分钟 K 线
启动所需历史蜡烛数: startup_candle_count = 800  # 需要 800 根 5 分钟 K 线的历史数据
止损: stoploss = -0.99                   # 名义止损设为 -99%（几乎不触发，因为策略自己管理止损）
```

### 关键配置建议（文件头部注释）：

| 参数 | 推荐值 | 含义 |
|------|--------|------|
| 同时开单数 | 6~12 个 | 同时持有的交易对数量 |
| 交易对数量 | 40~80 对 | 用交易量筛选的交易对列表 |
| 计价币种 | USDT/USDC | 推荐稳定币交易对，不推荐 BTC/ETH 交易对 |
| 黑名单 | 杠杆代币 | 必须屏蔽 *BULL, *BEAR 等杠杆代币 |
| 时间周期 | 必须 5m | 不可修改 |

## 四、10+ 种交易模式详解 (第 117~176 行)

这是策略最核心的设计之一 -- **每笔交易都有一个"模式标签"(tag)**，不同模式有不同的入场/出场/仓位管理逻辑。

### 做多 (Long) 模式:

| 模式名称 | 标签(tag) | 含义 | 通俗解释 |
|----------|-----------|------|----------|
| **Normal** | 1~13 | 常规模式 | "标准买入" -- 基于经典技术指标的入场 |
| **Pump** | 21~26 | 泵动模式 | "抓涨停" -- 在价格急速上涨(pump)时寻找入场机会 |
| **Quick** | 41~53 | 快速模式 | "快进快出" -- 寻找短期快速盈利的机会 |
| **Rebuy** | 61~63 | 重新买入 | "越跌越买" -- 典型的 DCA (定投/网格)策略 |
| **High Profit** | 81~82 | 高利润模式 | "大鱼" -- 等待高利润目标的交易 |
| **Rapid** | 101~110 | 极速模式 | "闪电战" -- 非常短线的交易 |
| **Grind** | 120 | 磨利模式 | "小步快跑" -- 通过频繁小额交易积累利润 |
| **BTC** | 121 | BTC专用 | 专门针对 BTC 的交易逻辑 |
| **Top Coins** | 141~145 | 头部币种 | 只在指定的头部币种上交易 |
| **Scalp** | 161~163 | 剥头皮 | "薅羊毛" -- 极短线微利交易 |

### 做空 (Short) 模式 (仅期货模式可用):

做空模式的标签从 500 开始，结构与做多完全对称，包含 Normal/Pump/Quick/Rebuy 等相同类型。

**标签的作用**：当策略决定买入时，会给这笔交易打上标签（比如 "1" 表示 Normal 模式条件1），之后的出场和仓位管理都会根据这个标签选择对应的逻辑。

## 五、技术指标体系 (第 2984~3700 行)

策略在 **5 个时间周期** 上计算指标，就像同时用 5 个不同倍率的望远镜观察市场：

### 5.1 多时间周期数据源

| 时间周期 | 函数 | 用途 |
|----------|------|------|
| **5m** (主周期) | `base_tf_5m_indicators()` | 精确入场/出场信号 |
| **15m** | `informative_15m_indicators()` | 短期趋势确认 |
| **1h** | `informative_1h_indicators()` | 中期趋势方向 |
| **4h** | `informative_4h_indicators()` | 大趋势判断 |
| **1d** | `informative_1d_indicators()` | 宏观趋势 / 超买超卖 |

此外，还会单独获取 **BTC 的数据** (`btc_info_*_indicators`)，因为 BTC 的走势对整个加密市场有统领作用。

### 5.2 使用的技术指标清单

在 5 分钟主周期 (`base_tf_5m_indicators`, 第 3537 行) 上计算的指标：

| 指标类别 | 具体指标 | 初学者解释 |
|----------|----------|------------|
| **趋势指标** | EMA (3/9/12/16/20/26/50/100/200) | 指数移动平均线，数字越大代表越长期的趋势 |
| | SMA (9/16/21/30/200) | 简单移动平均线 |
| **动量指标** | RSI (3/4/14/20) | 相对强弱指标。RSI<30 = 超卖(可能反弹)，RSI>70 = 超买(可能回调) |
| | StochRSI | 随机 RSI，RSI 的 RSI，更灵敏 |
| | Williams %R (14/480) | 类似 RSI 的超买超卖指标 |
| | ROC (2/9) | 价格变动率，衡量价格变化的速度 |
| | KST | 知道确定的事 (Know Sure Thing)，多周期动量合成指标 |
| | AROON (14) | 阿隆指标，判断趋势强度和方向 |
| **波动指标** | Bollinger Bands (20,2.0) & (40,2.0) | 布林带，价格在上下轨之间波动。跌破下轨 = 超卖信号 |
| **资金流指标** | MFI (14) | 资金流量指标，结合价格和成交量判断买卖压力 |
| | CMF (20) | 蔡金资金流，正值=资金流入，负值=资金流出 |
| | OBV | 能量潮，成交量累积方向 |

## 六、入场逻辑详解 (第 11889~25073 行)

入场逻辑在 `populate_entry_trend()` 方法 (第 11889 行) 中实现，这是策略中最庞大的部分。

### 6.1 入场流程

```
对每一个启用的入场信号条件:
  1. 检查保护条件 (Protections) -> 过滤掉危险的市场环境
  2. 检查入场条件 (Entry Logic) -> 判断是否满足买入条件
  3. 打上模式标签 (enter_tag) -> 记录这笔交易的类型
```

### 6.2 保护条件 (Protections) -- "安全网"

保护条件的目的是**避免在危险的市场环境下入场**。以 Condition #1 (Normal 模式) 为例：

```python
# 第 11949 行起
long_entry_logic.append(df["num_empty_288"] <= allowed_empty_candles_288)  # 空蜡烛不能太多(流动性检查)
long_entry_logic.append(df["protections_long_global"] == True)            # 全局保护开启

# 以下是大量的组合保护条件，逻辑是：
# "如果 A 指标很差，那么 B 指标必须还行" -- 即至少有一个指标说"还行"才能入场
```

**举例理解保护条件**：
```python
((df["RSI_3"] > 3.0) | (df["RSI_3_15m"] > 3.0) | (df["RSI_3_change_pct_1h"] > -50.0))
```
翻译成人话就是：
> "5分钟RSI(3)不能太低 **或者** 15分钟RSI(3)不能太低 **或者** 1小时RSI变化不能暴跌超过50%"
> -> 至少一个条件要满足，否则说明市场在**极端恐慌性抛售**，不应该入场。

每个 Condition 都有约 **100~200 条这样的保护条件**组合在一起（用 `&` 连接），形成一张非常严密的"安全网"。

### 6.3 入场信号 (Entry Logic) -- "扣扳机"

在通过所有保护条件后，策略会检查具体的入场信号。以 Condition #1 为例：

```python
# 第 12333 行起 - Logic
long_entry_logic.append(
    (df["EMA_26"] > df["EMA_12"])                              # 26期EMA在12期EMA上方(短期走弱)
    & ((df["EMA_26"] - df["EMA_12"]) > (df["open"] * 0.034))  # 两条EMA差距足够大(超卖够深)
    & ((df["EMA_26"].shift() - df["EMA_12"].shift()) > (df["open"] / 100.0))  # 前一根蜡烛也是这样
    & (df["close"] < (df["BBL_20_2.0"] * 0.999))              # 收盘价低于布林带下轨(严重超卖)
)
```
翻译成人话：
> "价格已经跌得很深了 (EMA 空头排列 + 跌破布林带下轨)，现在可能要反弹了，买入！"

这是一个经典的**均值回归**策略思路 -- 价格偏离平均值太多时，倾向于回归。

## 七、出场逻辑详解 (第 1761~43125 行)

出场逻辑在 `custom_exit()` 方法 (第 1761 行) 中触发，它根据入场时的标签(tag)分发到不同的出场函数。

### 7.1 出场流程

```
custom_exit() 被调用
  |
  +-- 根据 enter_tag 判断模式:
  |     +-- Normal 模式 -> long_exit_normal()
  |     +-- Pump 模式   -> long_exit_pump()
  |     +-- Quick 模式  -> long_exit_quick()
  |     +-- Rebuy 模式  -> long_exit_rebuy()
  |     +-- ... (10种模式各有对应出场函数)
  |
  +-- 每个出场函数内部有 4 层出场判断:
        +-- 1. long_exit_signals()    -- 原始卖出信号 (盈利时)
        +-- 2. long_exit_main()       -- 主要卖出信号 (盈利时)
        +-- 3. long_exit_williams_r() -- Williams %R 卖出 (盈利时)
        +-- 4. long_exit_dec()        -- 下降趋势卖出 (盈利时)
        +-- 5. long_exit_stoploss()   -- 止损卖出 (亏损时也执行)
```

### 7.2 止损机制 (第 43012 行)

策略有一个**分层止损系统**，而不是简单的固定止损：

| 止损类型 | 触发条件 | 说明 |
|----------|----------|------|
| **Doom Stop** | 亏损超过初始仓位的 12%~24% (现货) | "末日止损" -- 最后的安全网 |
| **U_E Stop** | 亏损 + 价格在 EMA200 下方 + CMF 为负 + RSI 回升 | "止损反弹出" -- 在小反弹时止损出场，减少损失 |

注意：策略设置 `stoploss = -0.99` (名义止损99%) 意味着它**不依赖 Freqtrade 的内置止损**，而是完全自己管理止损逻辑。

## 八、仓位管理系统 -- Grinding (磨利) 系统 (第 206~878 行配置，第 43150~51494 行实现)

这是策略最复杂也最精妙的部分。

### 8.1 什么是 Grinding？

**Grinding（磨利）** 是一种仓位管理技术：
- 当你持有一笔亏损的交易时，不是简单地等待或止损
- 而是在更低的价格**分批加仓**，降低平均持仓成本
- 然后在小幅反弹时**分批卖出加仓部分**获取小利润
- 通过多次这样的"磨"，逐步将整体亏损变为盈利或减小亏损

```
价格走势:        _______ (初始买入点)
                 \
                  \  <-- 价格下跌
                   + 加仓1 (降低均价)
                  /
                 /   <-- 小反弹
                 + 卖出加仓1 (赚取差价)
                  \
                   \  <-- 继续下跌
                    + 加仓2 (再次降低均价)
                   ... 如此反复 ...
```

### 8.2 Grinding 参数解读

以 `grind_1` 为例 (第 220~228 行)：

```python
grind_1_stop_grinds_spot = -0.50        # 如果磨利组亏损超50%，止损这组
grind_1_profit_threshold_spot = 0.018    # 磨利组盈利1.8%时，卖出获利
grind_1_stakes_spot = [0.24, 0.26, 0.28] # 第1/2/3次加仓分别用初始仓位的24%/26%/28%
grind_1_sub_thresholds_spot = [-0.12, -0.16, -0.20]  # 当价格下跌12%/16%/20%时触发加仓
```

策略配置了 **6 组** Grinding 参数 (grind_1 到 grind_6)，外加 **derisk (降风险)** 组，可以同时运行多组磨利操作。

### 8.3 三代仓位管理系统

策略维护了三个版本的仓位管理系统：

| 系统 | 函数 | 特点 |
|------|------|------|
| **v1** (原始) | `long_grind_adjust_trade_position()` | 最早的版本，用于 grind/btc 模式 |
| **v2** | `long_grind_adjust_trade_position_v2()` | 改进版，用于常规模式的旧交易 |
| **v3** (最新) | `long_grind_adjust_trade_position_v3()` | 最新版，用于 system_v3 之后的新交易 |

在 `adjust_trade_position()` (第 2511 行) 中根据交易的系统版本和模式标签分发到对应的管理函数。

### 8.4 Derisk (降风险)

Derisk 是 Grinding 的反面操作：
- 当整体亏损达到一定阈值时，**卖出一部分仓位**来降低风险敞口
- 例如：`regular_mode_derisk_1_spot = -0.24` 表示当现货亏损达到 24% 时触发降风险

## 九、入场确认与资金管理 (第 2345~2507 行, 第 11465~11577 行)

### 9.1 自定义仓位大小 -- `custom_stake_amount()` (第 2345 行)

不同模式使用不同的初始仓位比例：

| 模式 | 仓位倍数 | 意义 |
|------|----------|------|
| 常规模式 (system_v3_2) | `system_v3_2_stake_multiplier = 1.0` | 全额仓位 |
| 常规模式 (system_v3_1) | `system_v3_1_stake_multiplier = 0.50` | 半仓 (为后续加仓留空间) |
| Rebuy 模式 | `system_v3_rebuy_mode_stake_multiplier` | 较小仓位 (会频繁加仓) |
| Grind 模式 | 更小的乘数 | 小仓位频繁操作 |

### 9.2 入场确认 -- `confirm_trade_entry()` (第 11465 行)

在订单下达前的最后一道关卡：

1. **Grind 模式检查**：确认交易对在允许的 Grind 币种列表中，且 Grind 模式的开仓数未达上限
2. **Top Coins 模式检查**：确认交易对在头部币种列表中
3. **Scalp 模式检查**：确认有足够的空余交易槽位
4. **期货模式多/空仓位数量限制**
5. **滑点保护**：如果实际执行价格与信号价格偏差超过 `max_slippage`，取消订单

### 9.3 出场确认 -- `confirm_trade_exit()` (第 11581 行)

- 阻止 Freqtrade 的内置 `stop_loss` 和 `trailing_stop_loss`（策略自己管理止损）
- 支持 **Hold 功能**：通过 `hold-trades.json` 文件可以锁定特定交易不被卖出
- 在 `exit_profit_only` 模式下，只有盈利才允许卖出

## 十、BTC 参考机制 (第 3717~3840 行)

策略会单独获取 BTC 的行情数据，计算 BTC 专属指标，因为：

> BTC 是加密市场的"风向标"。当 BTC 暴跌时，大多数山寨币也会跟跌。策略通过监控 BTC 的指标来判断整体市场环境是否安全。

这些 BTC 指标会被合并到每个交易对的数据中，作为保护条件的一部分。

## 十二、策略核心逻辑流程总结

用一个完整的例子来说明策略的运作流程：

```
1. 每 5 分钟新蜡烛到来
   |
2. populate_indicators() 计算所有指标
   |  -> 5m/15m/1h/4h/1d 五个周期的技术指标
   |  -> BTC 的参考指标
   |
3. populate_entry_trend() 检查入场条件
   |  -> 遍历 160+ 个入场条件
   |  -> 如果 Condition #1 满足: 保护条件通过 + EMA空头排列 + 跌破布林带下轨
   |  -> 打上标签 enter_tag = "1" (Normal 模式)
   |
4. confirm_trade_entry() 最终确认
   |  -> 滑点检查通过
   |  -> 仓位数量检查通过
   |
5. custom_stake_amount() 计算仓位大小
   |  -> Normal 模式, system_v3_2: 使用全额仓位
   |
6. 订单成交，开始持仓
   |
7. 每 5 分钟 custom_exit() 检查是否出场
   |  -> 盈利时: 检查 exit_signals / exit_main / exit_williams_r / exit_dec
   |  -> 亏损时: 检查 stoploss (doom stop / u_e stop)
   |
8. 每 5 分钟 adjust_trade_position() 检查是否需要调整仓位
   |  -> 价格下跌了? 执行 Grinding 加仓
   |  -> 加仓部分盈利了? 卖出加仓部分获利
   |  -> 亏损太大? 执行 Derisk 减仓
   |
9. 最终出场
   -> 触发卖出信号，平仓获利/止损
```

## 十五、策略设计哲学总结

1. **均值回归为主**：大部分入场条件都是在价格**超卖**时买入，期待价格回归均值
2. **多重安全网**：100+ 条保护条件层层过滤，宁可错过也不要错买
3. **精细化仓位管理**：通过 Grinding 系统，将"一次交易"变成"多次微交易"的组合
4. **多时间周期确认**：同时观察 5 个时间周期，避免被单一周期的噪音误导
5. **BTC 市场感知**：始终关注 BTC 走势作为大盘风向标
6. **现货/期货双支持**：所有参数都有 spot 和 futures 两套配置
7. **极度防御性**：策略的大量代码都在做"什么时候不该交易"的判断，而不是"什么时候该交易"

这是一个**成熟的、以风控为核心的量化策略**，适合中长期稳定运行，而非追求暴利。
